# Web Docs — 文档技能工厂

给定文档站 URL，分析目录结构，生成一个**纯在线链接**的文档查阅技能（SKILL.md）。

> **依赖**：`defuddle` CLI（如未安装：`npm install -g defuddle`）

---

## 工作流程

### 第 1 步：确认基本信息

向用户确认：
- **文档站名称**（如 "React"、"Tailwind CSS"）
- **文档站首页/目录 URL**（入口页，用于分析结构）
- **技能名称**（kebab-case，如 `react-docs`）

### 第 2 步：确定技能存放路径

查看项目配置决定技能存放位置：

```bash
cat .pi/settings.json
```

默认项目技能路径为 `.pi/skills/`，生成的技能放在 `.pi/skills/<skill-name>/SKILL.md`。

### 第 3 步：分析文档结构

用 defuddle 提取首页/目录页，识别页面层级：

```bash
defuddle parse <文档站首页URL> --md
```

从提取内容中找出：
- **一级分类**（如"入门"、"核心概念"、"API 参考"）
- **二级页面**（每个分类下的具体页面及其 URL）
- **URL 规则**（相对/绝对路径、编码规则）

如果首页没有完整目录，找 `sitemap`、`sidebar`、`nav` 或 `/docs` 索引页。

### 第 4 步：生成 SKILL.md

按以下模板生成，**仅保留网页链接，不缓存本地内容**：

````markdown
---
name: <skill-name>
description: 按需访问 <文档站名称>（<文档站域名>）。当用户询问 <相关关键词> 时使用此技能，所有内容通过 defuddle 从官网提取。
---

# <文档站名称> 文档查阅

<一句话描述文档站>

> **所有文档均在线获取**，使用 `defuddle parse <url> --md` 提取纯净 Markdown。

## 查阅原则

1. **全部在线获取** — 使用 `defuddle parse <url> --md` 从官网提取
2. **从索引入手** — 从本文档表格找到对应页面 URL，再 defuddle 获取内容
3. **聚焦核心** — 提取后聚焦用户问题的具体段落
````

### 例外：本地文档

**仅当**该软件通过包管理器安装且自带本地文档时（如 `pi` 通过 Homebrew 安装在 Cellar），才在 SKILL.md 中增加「本地文档路径」章节，格式参考 `pi-docs`：

```markdown
> **本地文档路径**（自动跟随版本升级）：
> `/opt/homebrew/opt/<软件名>/...docs/`
```

其余情况一律不写本地路径，全部走在线链接。

---

## 生成后检查清单

- [ ] `name` 符合 kebab-case（1-64 字符，小写字母+数字+连字符）
- [ ] `description` 明确触发场景和关键词（≤1024 字符）
- [ ] 总览表格覆盖所有一级分类
- [ ] 所有 URL 是完整可访问的绝对路径
- [ ] URL 有特殊编码规则时在「查阅原则」中注明
- [ ] **没有写入不必要的本地缓存路径**（除非软件自带）
- [ ] 文件保存到 `.pi/skills/<skill-name>/SKILL.md`

---

## 快速命令速查

```bash
# 查看项目技能配置
cat .pi/settings.json

# 分析文档站首页结构
defuddle parse <文档站首页URL> --md

# 获取单个页面标题
defuddle parse <页面URL> -p title
```
